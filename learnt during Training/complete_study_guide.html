Chapter 2 — Java OOP + Collections
Core Java fundamentals — extremely high viva frequency
4 OOP Pillars
Encapsulation
Wrap data + methods. Private variables + public getters/setters. Like ATM — press buttons but can't see internals. In Selenium: private WebDriver driver;
Inheritance
Child gets parent properties via extends. BaseTest → LoginTest. Multiple inheritance NOT allowed with classes (diamond problem). Only with interfaces.
Polymorphism
Compile-time = overloading (same name, diff params). Runtime = overriding (child redefines parent). WebDriver driver = new ChromeDriver(); is runtime polymorphism.
Abstraction
Hide complexity, show only functionality. WebDriver is interface — says WHAT to do. ChromeDriver is class — says HOW. You call driver.click() without knowing internals.

Key Concepts — Quick Fire
Q== vs .equals()?
== compares memory address (same object?). .equals() compares content (same value?). ALWAYS use .equals() for Strings in Selenium assertions.
Qfinal, finally, finalize?
final = cannot change/override/extend. finally = always runs after try-catch (perfect for driver.quit()). finalize = called by GC before object destroyed.
Qstatic keyword?
Belongs to CLASS not object. Shared across all instances. Called without creating object. Driver is often static in BaseTest so all test classes share it.
Qthis vs super?
this = current class/object. super = parent class. super(driver) in LoginPage constructor calls BaseTest/BasePage constructor.
QString immutability?
Once created, cannot change. Any modification creates NEW String object. StringBuilder is mutable (not thread safe). StringBuffer is mutable + thread safe.

Exception Handling in Selenium
NoSuchElementException
Element not found on page. Fix: check locator, add wait.
StaleElementReferenceException
Element found, page refreshed, reference invalid. Fix: find element again.
ElementClickInterceptedException
Another element blocking click. Fix: JS click or close overlay first.
TimeoutException
Wait condition not met in time. Fix: increase timeout or fix locator.

Chapter 4 — Waits (All Types)
Most tested concept in every Selenium interview
⚡
Why waits? Modern websites use AJAX — content loads dynamically AFTER page opens. Selenium tries to find elements before they load → NoSuchElementException. Waits = patience for Selenium.
3 Types Comparison
TypeScopeTimingUse When
Thread.sleep(3000)Entire programFixed — ALWAYS waits 3s even if ready in 0.5s❌ NEVER in real code
Implicit WaitALL findElementsUp to X seconds, moves on when readyGeneral page loads
Explicit WaitONE specific elementUp to X seconds, for specific conditionDynamic content
Fluent WaitONE specific elementCustom polling + ignore exceptionsComplex dynamic elements
Code Examples
// Implicit Wait — set once, applies everywhere driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10)); // Explicit Wait — specific element + condition WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10)); wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("btn"))); wait.until(ExpectedConditions.elementToBeClickable(By.id("submit"))).click(); // Fluent Wait — custom polling + ignore exceptions FluentWait<WebDriver> fwait = new FluentWait<>(driver) .withTimeout(Duration.ofSeconds(30)) .pollingEvery(Duration.ofSeconds(2)) .ignoring(NoSuchElementException.class); WebElement el = fwait.until(d -> d.findElement(By.id("dynamicEl")));
All ExpectedConditions
Element Conditions
visibilityOfElementLocated — visible + present
presenceOfElementLocated — in DOM (even hidden)
elementToBeClickable — visible + enabled
invisibilityOfElementLocated — disappears (spinner)
Page/Browser Conditions
alertIsPresent — alert popup appears
titleContains("text") — title has text
urlContains("/path") — URL has path
frameToBeAvailableAndSwitchToIt — frame ready
numberOfElementsToBe(By, 5) — element count

Chapter 5 — Selenium Advanced
Actions, JS Executor, Screenshots, Alerts, Frames, Windows, Grid
Actions Class
🖱️
Why Actions? Regular click() just clicks. Actions class simulates REAL user behavior — hover, drag, right-click, keyboard combos. Needs .perform() to execute.
Actions actions = new Actions(driver); actions.moveToElement(element).perform(); // hover actions.contextClick(element).perform(); // right-click actions.doubleClick(element).perform(); // double-click actions.dragAndDrop(source, target).perform(); // drag & drop actions.sendKeys(Keys.CONTROL, "a").perform(); // Ctrl+A actions.keyDown(Keys.SHIFT).sendKeys("hello").keyUp(Keys.SHIFT).perform();
JavaScript Executor
⚡
Why JSExecutor? When Selenium native methods fail — hidden elements, overlays, forced scrolling. Cast WebDriver to JavascriptExecutor since WebDriver doesn't expose JS execution directly.
JavascriptExecutor js = (JavascriptExecutor) driver; // Click element (bypasses overlay) js.executeScript("arguments[0].click();", element); // Scroll to element js.executeScript("arguments[0].scrollIntoView(true);", element); // Scroll by pixels js.executeScript("window.scrollBy(0, 500);"); // Get value String val = (String) js.executeScript("return arguments[0].value;", el); // Highlight element (debugging) js.executeScript("arguments[0].style.border='3px solid red';", element);

Chapter 6 — POM + TestNG Complete
PageFactory, BaseTest, Annotations, Assertions, Listeners
POM — Why + How
Without POM ❌
findElement scattered everywhere. Change one locator = update 20 files. Messy, not reusable, hard to read.
With POM ✅
Each page = one class with all locators + actions. Change one locator = update ONE file. Clean, reusable, readable.
// LoginPage.java — Page class (locators + actions) public class LoginPage { @FindBy(id = "username") WebElement usernameField; @FindBy(id = "password") WebElement passwordField; @FindBy(css = "button[type='submit']") WebElement loginBtn; public LoginPage(WebDriver driver) { PageFactory.initElements(driver, this); // activates @FindBy } public void login(String user, String pass) { usernameField.sendKeys(user); passwordField.sendKeys(pass); loginBtn.click(); } } // LoginTest.java — Test class (only calls methods) loginPage = new LoginPage(driver); loginPage.login("tomsmith", "SuperSecretPassword!");
TestNG Annotation Order
📋
BeforeSuite → BeforeTest → BeforeClass → BeforeMethod → @Test → AfterMethod → AfterClass → AfterTest → AfterSuite
@BeforeMethod vs @BeforeClass
BeforeMethod = runs before EACH test → opens browser 5 times. BeforeClass = runs ONCE → all 5 tests share one browser. Use BeforeClass for efficiency.
Hard vs Soft Assert
Hard Assert — stops test immediately on fail. Soft Assert — continues, collects all, reports at end with soft.assertAll(). Use soft when checking multiple things.
TestNG Advanced Features
@Test(priority=1, groups="smoke", enabled=true, timeOut=5000) @Test(dependsOnMethods="testLogin") // skip if testLogin fails @DataProvider(name="loginData") public Object[][] data() { return new Object[][] { {"user1","pass1"}, {"user2","pass2"} }; } @Test(dataProvider="loginData") public void testLogin(String user, String pass) { ... } // Listeners — screenshot on failure public class MyListener implements ITestListener { public void onTestFailure(ITestResult result) { // take screenshot here } }

Aggregates + GROUP BY + HAVING
-- Aggregate functions SELECT COUNT(*), AVG(salary), SUM(salary), MAX(salary), MIN(salary) FROM employees; -- GROUP BY: count employees per department SELECT dept_id, COUNT(*) AS emp_count FROM employees GROUP BY dept_id; -- HAVING: filter groups (not WHERE!) SELECT dept_id, COUNT(*) FROM employees GROUP BY dept_id HAVING COUNT(*) > 2; -- WHERE filters rows, HAVING filters groups SELECT dept_id, AVG(salary) FROM employees WHERE hire_date > '2019-01-01' -- filter rows first GROUP BY dept_id HAVING AVG(salary) > 60000; -- filter groups after
RANK Functions — Window Functions
-- RANK: same salary = same rank, SKIPS next (1,2,2,4) SELECT name, salary, RANK() OVER (ORDER BY salary DESC) AS rnk FROM employees; -- DENSE_RANK: same salary = same rank, NO skip (1,2,2,3) SELECT name, salary, DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk FROM employees; -- ROW_NUMBER: unique, no ties (1,2,3,4 always) SELECT name, salary, ROW_NUMBER() OVER (ORDER BY salary DESC) AS rn FROM employees; -- Top earner per department SELECT * FROM ( SELECT name, salary, dept_id, ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rn FROM employees ) t WHERE rn = 1; -- 2nd highest salary SELECT MAX(salary) FROM employees WHERE salary < (SELECT MAX(salary) FROM employees);

Normalization
1NF
Atomic values. No repeating groups. Each row unique. No arrays in cells.
2NF
Must be 1NF. No partial dependency — non-key column depends on ENTIRE composite key.

Chapter 9 — BDD Cucumber
Behavior Driven Development, Gherkin, Feature files, Step Definitions
💡
What is BDD? Tests written in plain English that BUSINESS can read + understand. No technical jargon. Developer writes Java code behind each plain English step. Bridges gap between business and tech.
Gherkin Syntax — Feature File
# login.feature Feature: User Login As a registered user I want to login to the application So that I can access my account Scenario: Valid Login Given user is on the login page When user enters username "tomsmith" and password "SuperSecretPassword!" And user clicks the login button Then user should see "You logged into a secure area!" Scenario Outline: Login with multiple users Given user is on the login page When user enters username "<username>" and password "<password>" Then user should see "<message>" Examples: | username | password | message | | tomsmith | SuperSecretPassword!| secure area | | wronguser | wrongpass | username invalid|
Step Definitions — Java Code
// LoginSteps.java — maps Gherkin to Java public class LoginSteps { WebDriver driver; LoginPage loginPage; @Given("user is on the login page") public void userOnLoginPage() { driver = new ChromeDriver(); driver.get("https://the-internet.herokuapp.com/login"); loginPage = new LoginPage(driver); } @When("user enters username {string} and password {string}") public void enterCredentials(String username, String password) { loginPage.enterUsername(username); loginPage.enterPassword(password); } @Then("user should see {string}") public void verifyMessage(String expected) { String actual = loginPage.getFlashMessage(); Assert.assertTrue(actual.contains(expected)); } }

Chapter 12 — Docker + GenAI Basics
Containerization, Selenium Grid in Docker, AI concepts
Docker — Key Concepts
What is Docker?
Containerization platform. Packages app + dependencies into a container. Runs the same everywhere — dev, staging, production. "Works on my machine" problem solved.
Image vs Container
Image = blueprint/template (like a class). Container = running instance of image (like an object). Containers are isolated, lightweight.
Docker + Selenium Grid
Run Selenium Grid inside Docker containers. Each browser = separate container. Consistent, clean environment for every test run. No driver version conflicts.

GenAI + GitHub Copilot
What is Generative AI?
AI that can CREATE new content — text, code, images. Based on Large Language Models (LLMs). Examples: ChatGPT, Gemini, Claude. In testing: generate test cases, locators, step definitions.
GitHub Copilot
AI coding assistant that lives in your IDE. Suggests code as you type. Can generate: Selenium scripts, XPath locators, TestNG tests, step definitions. Powered by OpenAI Codex.
Prompt Engineering

Responsible AI
Use AI ethically. Don't feed sensitive customer data to AI tools. Verify AI-generated code before using. AI can make mistakes — always review output.

Chapter 13 — Complete Viva Q&A Master List
Every question that can be asked — know these cold
Testing Theory
QWhat is Selenium?
Open source tool to automate web browser testing. Supports Java, Python, C#. Components: IDE, RC (deprecated), WebDriver, Grid.
QVerification vs Validation?
Verification = building it RIGHT (process check, no execution). Validation = building the RIGHT thing (actual testing with execution).
QSmoke vs Sanity testing?
Smoke = broad shallow check if build is stable enough for testing. Sanity = narrow deep check of specific feature after bug fix.
QSeverity vs Priority?
Severity = technical impact of bug on system. Priority = business urgency to fix. Logo missing = low severity, high priority.
QQA vs QC?
QA = process oriented, prevent defects, proactive. QC = product oriented, find defects, reactive.
Selenium Questions
Qclose() vs quit()?
close() = closes CURRENT tab only. quit() = closes ENTIRE browser + kills WebDriver process. Always use quit() at end of test.
QfindElement vs findElements?
findElement = single WebElement, throws NoSuchElementException if not found. findElements = List<WebElement>, returns empty list (no exception) if not found.
QCSS vs XPath?
CSS is faster but can only traverse DOWNWARD in HTML. XPath is slower but can go in ANY direction — up, down, sideways using axes.
QImplicit vs Explicit wait?
Implicit = set once globally, applies to ALL findElements, waits up to X seconds. Explicit = for SPECIFIC element, waits for specific condition using ExpectedConditions. Don't use both together — can conflict.
QWhat is StaleElementReferenceException?
Element was found, then page refreshed/DOM changed, old reference is now invalid. Fix: find element again fresh.
QWhat is ElementClickInterceptedException?
Another element (overlay, cookie banner) is blocking the click on target element. Fix: close overlay, scroll to element, or use JavascriptExecutor click.
QWhy use JavascriptExecutor?
When native Selenium fails — hidden elements, overlays blocking clicks, need to scroll precisely, or get/set values JS way. Must cast WebDriver: (JavascriptExecutor) driver.
QWhat is headless browser?
Browser runs without visible window. Used in CI/CD servers with no display. Faster execution, lower memory. Set with ChromeOptions: --headless=new.
Qget() vs navigate().to()?
Both open URL. navigate().to() maintains browser history so back/forward work. get() does not maintain history.
Java Questions
QJDK vs JRE vs JVM?
JVM = runs bytecode (engine). JRE = JVM + libraries to run Java programs. JDK = JRE + compiler + tools to develop Java. Developers need JDK.
Q== vs .equals()?
== compares memory reference (same object?). .equals() compares content (same value?). Always use .equals() for Strings in assertions.
QMethod overloading vs overriding?
Overloading = same name, different parameters, same class — compile time polymorphism. Overriding = child redefines parent method — runtime polymorphism.
QWhy can't Java have multiple inheritance with classes?
Diamond problem — if B and C both have method show() and D extends both, which show() does D get? Ambiguous. Java solves this by allowing multiple interface implementation.
POM + TestNG Questions
QWhat is POM?
Page Object Model — design pattern where each web page has its own Java class with locators and action methods. Tests just call page methods, no findElement in test classes.
QWhat is PageFactory?
Enhanced POM using @FindBy annotations. PageFactory.initElements(driver, this) activates them. Lazy initialization — element searched only when actually used.
Q@BeforeMethod vs @BeforeClass?
BeforeMethod = runs before EACH test, opens browser multiple times. BeforeClass = runs ONCE before all tests in class, all tests share one browser. BeforeClass is more efficient.
QHard vs Soft Assert?
Hard Assert stops test immediately on first failure. Soft Assert continues test, collects all failures, reports all at end with soft.assertAll(). Use soft when checking multiple validations.
QWhat is TestNG Listener?
Class implementing ITestListener that automatically reacts to test events. Most important: onTestFailure — take screenshot automatically when test fails.
QWhat is dependsOnMethods?
If parent test fails, dependent tests are automatically skipped. Saves time and avoids misleading failures. TC02 skipped if TC01 fails.
SQL Questions
QWHERE vs HAVING?
WHERE filters individual ROWS before grouping. HAVING filters GROUPS after GROUP BY. WHERE cannot use aggregate functions, HAVING can.
QRANK vs DENSE_RANK vs ROW_NUMBER?
RANK skips after ties (1,2,2,4). DENSE_RANK no skipping (1,2,2,3). ROW_NUMBER always unique even for equal values (1,2,3,4).
QTRUNCATE vs DELETE vs DROP?
TRUNCATE = all rows, DDL, no rollback, fast, resets auto-increment. DELETE = specific/all rows, DML, rollbackable, slow. DROP = removes entire table permanently.
QWhat is normalization?
Organizing tables to reduce data redundancy. 1NF = atomic values. 2NF = no partial dependency. 3NF = no transitive dependency.
Framework + Tools
QWhat is WebDriverManager?
Library that auto-downloads matching browser driver at runtime. No manual chromedriver.exe setup. Detects Chrome version, downloads correct driver automatically.
QWhat is Apache POI?
Java library to read/write Microsoft Excel files. Used for data-driven testing — read test data from Excel, write results back. XSSFWorkbook for .xlsx files.
QWhat is ExtentReports?
Third-party library for beautiful HTML test reports with charts, screenshots, logs. Created in Listeners. flush() must be called to write everything to HTML file.
QWhat is BDD?
Behavior Driven Development. Tests written in plain English (Gherkin) that business can read. Given-When-Then format. Cucumber is the popular BDD framework for Java.
QWhat is ThreadLocal?
Gives each parallel thread its own private WebDriver instance. Without it, threads share one driver and interfere with each other causing random failures.
QWhat is Jenkins?
CI/CD tool that automatically runs tests on every code commit. Detects changes in Git, runs mvn test, generates reports, emails results to team.
QWhat is Docker?
Containerization platform. Packages app + all dependencies into portable container. Solves "works on my machine" problem. Used with Selenium Grid for consistent test environments.
