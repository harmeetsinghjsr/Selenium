Selenium WebDriver — Basics
Architecture · Browser control · WebElement interactions · Exceptions
Q1–Q20
1
What is Selenium WebDriver? How is it different from Selenium RC?🔥 HOT
▼
Selenium WebDriver is a browser automation library that directly controls the browser through browser-specific drivers (ChromeDriver, GeckoDriver) using the W3C protocol. Unlike Selenium RC, WebDriver does NOT require a separate server — it communicates directly with the browser. RC was slow (needed a Java server as proxy), WebDriver is fast and supports all modern browsers. RC is now deprecated; WebDriver is the standard.
2
Explain the Selenium WebDriver architecture.🔥 HOT
▼
Flow: Your Java code → Selenium WebDriver API → Browser Driver (ChromeDriver/GeckoDriver) → Actual Browser.

WebDriver is an interface. ChromeDriver is the class that implements it. The driver receives JSON Wire Protocol/W3C commands from your test and translates them into native browser actions. Selenium 4 uses W3C WebDriver standard — all browsers follow the same protocol, making behavior consistent.
3
What is the difference between driver.close() and driver.quit()?🔥 HOT
▼
close() closes only the current active tab/window. If you have 3 windows open, it closes only the focused one.
quit() closes ALL windows and tabs and completely kills the WebDriver session and browser process.

Always use quit() in @AfterMethod/@AfterClass. Using close() without switching windows first may leave orphan ChromeDriver processes running in background.
4
What is the difference between driver.get() and driver.navigate().to()?
▼
Both open a URL in the browser. The difference:
• driver.get(url) — opens URL, waits for page load, does NOT maintain browser history
• driver.navigate().to(url) — opens URL AND maintains browser history

Practical impact: navigate().back() and navigate().forward() work correctly only when you used navigate().to(). If you used get(), back/forward may not work as expected.
5
What is findElement() vs findElements()?🔥 HOT
▼
findElement(By locator) — Returns a single WebElement. Throws NoSuchElementException if no element matches.
findElements(By locator) — Returns a List<WebElement>. Returns empty list (no exception) if no elements match.

Use findElements() when: checking if element exists without try-catch, getting all links on page, counting rows in a table. Check list.size() == 0 to verify absence.
6
What is StaleElementReferenceException? How do you fix it?🔥 HOT
▼
Thrown when a WebElement reference becomes stale — the element was found, then the page DOM changed (page refresh, AJAX update, navigation), and the old reference no longer points to a valid element.

Fixes:
1. Find the element again fresh: re-call driver.findElement()
2. Use explicit wait for element to reappear: wait.until(ExpectedConditions.stalenessOf(el)) then find again
3. Avoid storing element references if page refreshes between uses
7
What is ElementClickInterceptedException? How do you resolve it?🔥 HOT
▼
Thrown when Selenium tries to click an element but another element is on top of it, intercepting the click. Common causes: cookie banners, modals, loading overlays, sticky headers.

3 Fixes:
1. Close the blocking element first: driver.findElement(By.id("overlay")).click()
2. Scroll element into view: js.executeScript("arguments[0].scrollIntoView(true);", element)
3. JavaScript click (bypasses DOM completely): js.executeScript("arguments[0].click();", element)
8
What is NoSuchElementException and when does it occur?
▼
Thrown when Selenium cannot find any element matching the given locator in the current DOM. Causes: wrong locator (typo in id/xpath), element inside an iframe (forgot to switch), element not yet loaded (need a wait), element is inside a Shadow DOM, page has not navigated yet. Fix: verify locator in DevTools, add explicit wait, switch to correct frame first.
9
What is WebDriverWait and how does it differ from Thread.sleep()?🔥 HOT
▼
Thread.sleep(3000) suspends the thread for exactly 3 seconds — even if element becomes available in 500ms, it still waits the full 3 seconds. Wastes time, makes tests slow.

WebDriverWait polls every 500ms and proceeds AS SOON AS the condition is met. Max wait = whatever you set (e.g., 10 seconds). If element ready in 1 second, it moves on in 1 second. Intelligent, efficient, industry standard.
10
How do you handle alerts/popups in Selenium?
▼
Switch to the alert first using driver.switchTo().alert(), then interact:
• alert.getText() — get message text
• alert.accept() — click OK
• alert.dismiss() — click Cancel
• alert.sendKeys("text") — type into prompt dialog

Types: Simple alert (info, just OK), Confirm (OK + Cancel), Prompt (text input). Always handle alert before interacting with page elements, otherwise you'll get UnhandledAlertException.
11
How do you handle multiple browser windows/tabs?
▼
1. Save parent window: String parent = driver.getWindowHandle()
2. Get all windows: Set<String> all = driver.getWindowHandles()
3. Loop to find the new window: for(String w : all) { if(!w.equals(parent)) driver.switchTo().window(w); }
4. Do work in child window
5. Close child: driver.close()
6. Switch back: driver.switchTo().window(parent)

Key point: getWindowHandle() = current window (String). getWindowHandles() = all windows (Set).
12
How do you switch to a frame/iframe and back?
▼
Switch INTO frame:
• driver.switchTo().frame("frameId") — by id or name
• driver.switchTo().frame(0) — by index (0-based)
• driver.switchTo().frame(webElement) — by WebElement reference

After switching, all findElement() calls look inside the frame only.

Switch BACK:
• driver.switchTo().defaultContent() — back to main page (out of all frames)
• driver.switchTo().parentFrame() — up one level (for nested frames)
13
What is JavascriptExecutor? When do you use it?🔥 HOT
▼
JavascriptExecutor is an interface that allows running JavaScript code in the browser from Selenium. Must cast WebDriver: JavascriptExecutor js = (JavascriptExecutor) driver

Use when:
• Element click intercepted by overlay → js.executeScript("arguments[0].click();", el)
• Scroll to element → js.executeScript("arguments[0].scrollIntoView(true);", el)
• Scroll page → js.executeScript("window.scrollBy(0,500);")
• Get/set hidden values → js.executeScript("return arguments[0].value;", el)
• Highlight element for debugging → set border via JS
14
What is the Actions class and what can it do?
▼
Actions class simulates complex user interactions that go beyond basic click/type — real mouse and keyboard behavior.

• moveToElement(el) — mouse hover (reveals dropdown menus)
• contextClick(el) — right-click
• doubleClick(el) — double click
• dragAndDrop(source, target) — drag and drop
• sendKeys(Keys.CONTROL, "a") — keyboard combos
• keyDown(Keys.SHIFT).sendKeys("text").keyUp(Keys.SHIFT)

Always call .perform() at end to execute the built action chain.
15
How do you take a screenshot in Selenium?
▼
TakesScreenshot ts = (TakesScreenshot) driver — cast WebDriver to TakesScreenshot interface.
File src = ts.getScreenshotAs(OutputType.FILE) — capture screenshot.
FileUtils.copyFile(src, new File("screenshots/test.png")) — save to disk.

In Selenium 4: element.getScreenshotAs(OutputType.FILE) — element-level screenshot.

Best practice: Take screenshot in onTestFailure() listener method. Name file with test name + timestamp so files don't overwrite each other.
16
What is a headless browser? Why use it?
▼
A headless browser runs without a visible graphical window — the browser executes entirely in memory.

How: ChromeOptions options = new ChromeOptions(); options.addArguments("--headless=new"); driver = new ChromeDriver(options);

Why use it:
• CI/CD servers (Jenkins/Docker) have no display
• Faster execution (no rendering overhead)
• Lower memory usage
• Run many parallel instances simultaneously

When NOT to use: Debugging (can't see what's happening), testing visual UI appearance.
17
What is WebDriverManager and why is it needed?
▼
WebDriverManager is a library by Boni García that automatically downloads and configures the correct browser driver at runtime.

Without it: Manually download chromedriver.exe, match Chrome version, set system property path. When Chrome auto-updates, driver breaks.

With it: Just one line: WebDriverManager.chromedriver().setup() — detects your Chrome version, downloads matching driver, sets everything up.

Supports: Chrome, Firefox, Edge, Safari, Opera. Essential for CI/CD environments.
18
What is Shadow DOM? How does Selenium 4 handle it?
▼
Shadow DOM is an encapsulation feature in modern web components. Elements inside Shadow DOM are hidden from regular DOM queries — normal findElement() cannot see them.

Selenium 4 solution:
WebElement host = driver.findElement(By.id("shadowHost"));
SearchContext shadow = host.getShadowRoot();
WebElement el = shadow.findElement(By.cssSelector("input"));

Only CSS selectors work inside Shadow DOM — XPath does NOT work.
19
How do you handle dropdowns in Selenium?
▼
For <select> HTML dropdowns use the Select class:
Select sel = new Select(driver.findElement(By.id("dropdown")));

3 selection methods:
• sel.selectByVisibleText("Option 1") — by displayed text
• sel.selectByValue("opt1") — by value attribute
• sel.selectByIndex(0) — by position (0 = first)

Other methods: sel.getOptions(), sel.getFirstSelectedOption(), sel.getAllSelectedOptions(), sel.isMultiple()

For custom dropdowns (not <select>): click to open, then click options using regular locators.
20
What does driver.manage().window().maximize() do and when should you call it?
▼
Maximizes the browser window to the full screen size. Should be called immediately after creating the driver, before navigating to any URL.

Why important: Many elements are only visible at full screen (responsive design hides them on small screens). Screenshots capture full page. Element coordinates for Actions class are screen-based. Not maximizing can cause ElementNotInteractableException for elements outside viewport.

Alternative: driver.manage().window().setSize(new Dimension(1920, 1080)) for specific size.

Waits — Synchronization
Implicit · Explicit · Fluent · ExpectedConditions
Q21–Q30
21
Explain all types of waits in Selenium. Which is best?🔥 HOT
▼
Thread.sleep(ms) — Fixed pause. Program stops completely. Even if element ready in 1s, waits full time. ❌ Never use in production.

Implicit Wait — Set once globally, applies to ALL findElement() calls. Waits UP TO X seconds for element to appear. driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10))

Explicit Wait — For SPECIFIC element + specific condition. WebDriverWait + ExpectedConditions. Most precise. ✅ Industry standard.

Fluent Wait — Extended explicit wait. Custom polling interval + exception ignoring. Most flexible.

Best practice: Use Implicit for baseline + Explicit for dynamic elements. Never mix Implicit + Explicit on same element (can cause double-wait).
22
What is Fluent Wait and how is it different from Explicit Wait?
▼
Both are condition-based waits. Differences:

Explicit Wait: Default polling every 500ms. Throws exception immediately when NoSuchElementException occurs.

Fluent Wait: You control polling interval (e.g., every 2s). You specify which exceptions to IGNORE during polling (e.g., ignore NoSuchElementException so it keeps retrying).

FluentWait<WebDriver> wait = new FluentWait<>(driver).withTimeout(Duration.ofSeconds(30)).pollingEvery(Duration.ofSeconds(2)).ignoring(NoSuchElementException.class);

Use Fluent Wait for elements that appear intermittently or take long to load.
23
Can you use Implicit and Explicit waits together?🔥 HOT
▼
Not recommended. When used together they can conflict and cause unexpected behavior. Example: Implicit wait = 10s, Explicit wait = 5s. The explicit wait may wait up to 10s (implicit) + 5s (explicit) = 15s instead of 5s.

Best practice: Set implicit wait for general stability, and use explicit only for specific complex elements. Avoid both on the same element. Many teams use only explicit waits for full control.
24
Name 5 important ExpectedConditions with use cases.
▼
1. visibilityOfElementLocated(By) — element is in DOM AND visible on page. Use for elements that appear after AJAX.
2. elementToBeClickable(By) — element is visible AND enabled. Use before clicking buttons/links.
3. invisibilityOfElementLocated(By) — element disappears. Use to wait for loading spinner to go away.
4. alertIsPresent() — JavaScript alert has appeared. Use before switching to alert.
5. frameToBeAvailableAndSwitchToIt(By) — frame loaded AND switches to it. Use before interacting with frame content.
6. numberOfElementsToBe(By, count) — wait until X elements exist. Use for table rows loaded.
25
Why should you never use Thread.sleep() in Selenium tests?
▼
1. Wastes time — always waits the full duration even if element is ready in 100ms
2. Unreliable — if network is slow, even 3 seconds may not be enough, test fails
3. Slows CI/CD — 100 tests × 3s sleep = 5+ minutes wasted just sleeping
4. Brittle — works today, fails tomorrow when server is busy
5. Not maintainable — magic numbers scattered in code

Always use WebDriverWait with ExpectedConditions — it exits as soon as condition is met and fails fast with clear error if timeout exceeded.
26
What is presenceOfElementLocated vs visibilityOfElementLocated?
▼
presenceOfElementLocated — element is present in the DOM (HTML source) but may be hidden (display:none, visibility:hidden, opacity:0)

visibilityOfElementLocated — element is present in DOM AND visible on screen (non-zero dimensions, not hidden)

Use presence when you just need the element to exist (e.g., to get its attribute).
Use visibility when you need to interact with it (click, type).

Trying to click a present-but-hidden element throws ElementNotInteractableException.

Locators — XPath & CSS Deep Dive
All 8 locators · XPath axes · Dynamic XPath · CSS selectors
Q27–Q38
27
What are the 8 locators in Selenium? Which is most reliable?🔥 HOT
▼
1. By.id — ⭐ Most reliable, fastest. Always first choice.
2. By.name — when id not available
3. By.className — when unique class exists
4. By.tagName — for getting all elements of a type
5. By.linkText — exact text of <a> tag
6. By.partialLinkText — partial text of <a> tag
7. By.cssSelector — ⭐ Fast, flexible, goes downward only
8. By.xpath — ⭐ Most powerful, any direction, slowest

Priority order: id > name > cssSelector > xpath. Avoid tagName/className when multiple elements share them.
28
What is the difference between CSS Selector and XPath?🔥 HOT
▼
CSS Selector: Can only traverse HTML tree downward (parent → child). Faster. Supported by all browsers natively. Syntax: tagName#id, .className, [attribute='value'], tr:nth-child(2)

XPath: Can traverse in ANY direction — up, down, sideways using axes. Slower. Can locate by text content: text()='Login'. Supports functions: contains(), starts-with(), last()

When to use XPath: When CSS can't locate element (need to go up to parent, need to find by text, complex sibling navigation).
29
What is absolute XPath vs relative XPath?
▼
Absolute XPath: Starts from root /html/body/div/table/tr/td/input. Uses single slash. Brittle — breaks if ANY element in the path changes. Never use in real projects.

Relative XPath: Starts from anywhere with double slash //input[@id='username']. Searches entire DOM. More stable, maintainable. Always preferred.

Additional: //input[@type='text'][1] — first matching. (//input[@type='text'])[2] — second in collection.
30
Explain XPath contains(), starts-with(), and text() functions.🔥 HOT
▼
contains(@attribute, 'value') — matches if attribute CONTAINS the value. Useful for dynamic IDs: //input[contains(@id,'username')] matches id="username123"

starts-with(@attribute, 'value') — matches if attribute STARTS WITH value. For IDs like "field_12345": //input[starts-with(@id,'field_')]

text()='Login' — matches by exact visible text content: //button[text()='Login']
contains(text(),'Login') — partial text match: //button[contains(text(),'Log')]

These 3 are critical for dynamic web apps where IDs/classes change.
31
What are XPath axes? Give a real example.🔥 HOT
▼
XPath axes navigate relative to a reference node. Critical when no id/class on target element.

Real scenario: Table with rows. Find "Edit" button in the row for "John":
//td[text()='John']/following-sibling::td/button

Key axes:
• parent:: — go UP one level
• ancestor:: — any level up
• child:: — direct children
• descendant:: — all levels down
• following-sibling:: — siblings AFTER current node (same parent)
• preceding-sibling:: — siblings BEFORE current node
32
Write XPath to find the 3rd row of a table.
▼
Several ways:
1. (//table//tr)[3] — 3rd tr anywhere
2. //table//tr[3] — 3rd tr inside table
3. //table/tbody/tr[3] — 3rd tr inside tbody
4. CSS equivalent: table tr:nth-child(3)

Note: If there's a header row (thead), data row 1 = tr[2]. Always inspect HTML to check if table has thead/tbody separation.
33
What are dynamic XPaths and why are they needed?
▼
Dynamic XPath is needed when element attributes change on every page load (e.g., id="user_1234567" where number is session-based).

Techniques:
1. contains(): //input[contains(@id,'user_')]
2. starts-with(): //input[starts-with(@id,'user')]
3. Text: //button[text()='Submit']
4. Axes: Navigate from a stable parent element
5. Position: (//input[@type='text'])[2]
6. Multiple attributes: //input[@type='text' and @placeholder='Email']
34
Write CSS selectors for: id, class, attribute, nth-child, child combinator.
▼
• ID: #username or input#username
• Class: .form-control or input.form-control
• Attribute: input[type='text'], input[name='email']
• Direct child only: div > input (input must be immediate child of div)
• Any descendant: div input (input anywhere inside div)
• nth-child: tr:nth-child(3), tr:nth-child(odd), tr:first-child, tr:last-child
• Multiple: input[type='text'][name='user']
• Contains class: [class*='input']
35
What are Relative Locators in Selenium 4?
▼
New in Selenium 4. Find elements relative to other elements — how humans describe things naturally.

RelativeLocator.with(By.tagName("input")).above(passwordField) — input above password
RelativeLocator.with(By.tagName("label")).toLeftOf(usernameField) — label left of username
RelativeLocator.with(By.tagName("button")).toRightOf(cancelBtn) — button right of cancel
RelativeLocator.with(By.tagName("label")).near(emailField) — within 50px
RelativeLocator.with(By.tagName("input")).below(usernameField)

Useful when traditional locators fail and layout positioning is consistent.
36
How do you verify if a checkbox is selected?
▼
WebElement checkbox = driver.findElement(By.id("agree"));
boolean isChecked = checkbox.isSelected(); — returns true if checked

To check: if(!checkbox.isSelected()) checkbox.click(); — only click if not already checked
To uncheck: if(checkbox.isSelected()) checkbox.click();

Same applies to radio buttons. Use isSelected() to verify state, not getAttribute("checked") which can be null even when checked.
37
What is chained locators in Selenium?
▼
Finding an element within a specific parent element, making searches more precise. Instead of searching entire page, first find the container, then find element within it.

WebElement form = driver.findElement(By.id("loginForm"));
WebElement username = form.findElement(By.name("user"));

This is safer when same locator exists in multiple places (e.g., multiple forms on one page). The second findElement() is called on the WebElement, not on driver — so it searches only within that element's DOM tree.
38
How do you find all broken links on a page using Selenium?
▼
1. Get all anchor tags: List<WebElement> links = driver.findElements(By.tagName("a"));
2. For each link, get href: String url = link.getAttribute("href");
3. Send HTTP request to each URL using HttpURLConnection
4. Check response code: 200 = OK, 404 = broken, 500 = server error

A link with no href or empty href is also broken. This is a common interview coding exercise in Selenium.

39
What is Selenium Grid? Explain Hub and Node.
▼
Selenium Grid allows running tests on multiple machines and browsers simultaneously.

Hub: Central server (runs on port 4444). Receives test requests. Routes tests to appropriate Nodes based on capability requested (browser type, version, OS). Only one Hub.

Node: Worker machine with actual browser installed. Registers with Hub on startup. Waits for tests. Executes when assigned. Can have multiple browsers. Multiple Nodes possible.

Selenium 4 Grid: Simplified setup. Standalone mode (Hub + Node together). Docker support built-in. Web UI at localhost:4444 to monitor.
40
How do you run tests in parallel using TestNG?
▼
In testng.xml: <suite name="Suite" parallel="tests" thread-count="3">

Parallel options:
• parallel="tests" — each <test> block in parallel
• parallel="classes" — each class in parallel
• parallel="methods" — each @Test method in parallel
• parallel="instances" — each instance in parallel

Critical: When parallel, each thread needs its OWN driver. Use ThreadLocal<WebDriver> — gives each thread private driver. Without ThreadLocal, threads share one driver → race conditions → random failures.
41
What is ThreadLocal? Why is it used in Selenium parallel testing?
▼
ThreadLocal<T> is a Java class that provides each thread with its own private instance of a variable. Think of it as a locker room — each person (thread) has their own locker (WebDriver).

private static ThreadLocal<WebDriver> driver = new ThreadLocal<>();
driver.set(new ChromeDriver()); // each thread sets its own
driver.get(); // each thread gets its own
driver.remove(); // cleanup after test

Without it: Thread-1 and Thread-2 share same driver object → Thread-1 navigates to login, Thread-2 navigates to checkout simultaneously → chaos.

2. --incognito — private mode, no cookies/history
3. --disable-notifications — block "Allow notifications?" popup
4. --start-maximized — open full screen

43
How do you handle cookies in Selenium?
▼
Cookies store session info, preferences, login state.

driver.manage().addCookie(new Cookie("name", "value")) — add cookie
driver.manage().getCookies() — get all cookies (Set<Cookie>)
driver.manage().getCookieNamed("sessionId") — get specific
driver.manage().deleteAllCookies() — delete all
driver.manage().deleteCookie(cookie) — delete specific

Use case: Skip login by adding session cookie → navigate directly to dashboard without going through login page. Saves test time. Login once, add cookie, all subsequent tests start logged in.
44
What is getPageSource() and when do you use it?
▼
driver.getPageSource() returns the complete HTML source of the current page as a String.

Use cases:
1. Check if text exists without using findElement (no exception risk): driver.getPageSource().contains("Welcome")
2. Scraping page content
3. Debugging — see exactly what DOM contains
4. Verify elements that may be dynamically generated

Note: Returns HTML at time of call, not initial page load. Dynamic content loaded via JS IS included if already executed.
45
What is the difference between isDisplayed(), isEnabled(), and isSelected()?
▼
isDisplayed() — Is element visible on screen? Returns false if hidden (CSS display:none, visibility:hidden). True if visible with non-zero dimensions.

isEnabled() — Is element enabled/interactive? Returns false if disabled (grayed out button, disabled input field). True if element can be interacted with.

isSelected() — Is element checked/selected? Only for checkboxes, radio buttons, option tags. Returns true if checkbox is checked or radio button is selected.

All return boolean. Throw NoSuchElementException if element doesn't exist (unlike findElements).
46
What is RemoteWebDriver and when is it used?
▼
RemoteWebDriver is a class that extends WebDriver to run tests on a remote machine or cloud instead of local machine.

URL hubUrl = new URL("http://localhost:4444");
ChromeOptions options = new ChromeOptions();
WebDriver driver = new RemoteWebDriver(hubUrl, options);

Used for: Selenium Grid (run tests on Hub/Node setup), Cloud testing (BrowserStack, SauceLabs — just change URL to their cloud hub), Docker Grid. The test code is identical — only the driver creation changes.

Software Testing Theory

Q47–Q66
47
What is the difference between Verification and Validation?🔥 HOT
▼
Verification = "Are we building the product RIGHT?" — Checks the process and documents. No software execution. Methods: reviews, walkthroughs, inspections. Done early (requirements, design phase).

Validation = "Are we building the RIGHT product?" — Checks if final product meets customer needs. Involves actual software execution. Methods: functional testing, UAT. Done later.

Simple: Verification checks PROCESS. Validation checks PRODUCT.
48
What is the difference between Severity and Priority?🔥 HOT
▼
Severity = Technical impact of the bug on the system. Set by QA engineer.
Priority = Business urgency of fixing the bug. Set by Product Manager/Business.

Classic examples:
• Company logo missing on homepage → Low Severity (doesn't break functionality) + High Priority (brand image — fix ASAP)
• Data corruption in rarely used admin feature → High Severity + Low Priority (critical but nobody uses it daily)
• Login broken → High Severity + High Priority (nobody can use the app)
49
Explain the complete Defect/Bug Life Cycle.🔥 HOT
▼
New → Bug raised by QA tester
Assigned → Given to developer by lead
Open → Developer starts working on it
Fixed → Developer says it's fixed, build deployed
Retest → QA verifies the fix
Closed → QA confirms fix works

Alternative paths:
→ Rejected — Developer says it's not a bug (working as designed)
→ Duplicate — Same bug already reported
→ Deferred — Valid bug, fix pushed to next release
→ Reopen — Fix didn't work, back to developer
50
What is STLC? Explain all phases.
▼
Software Testing Life Cycle — phases of testing process:

1. Requirement Analysis — QA reads requirements, identifies testable items, raises queries for unclear points
2. Test Planning — Test Manager defines scope, resources, timeline, risk assessment. Creates Test Plan document
3. Test Case Design — Write test cases, prepare test data
4. Environment Setup — Configure servers, browsers, Selenium setup
5. Test Execution — Run test cases, log defects for failures
6. Test Closure — Test summary report, lessons learned, metrics
51
What is the difference between Smoke and Sanity testing?🔥 HOT
▼
Smoke Testing: Broad, shallow testing to check if a new build is stable enough for detailed testing. Tests basic, critical functionalities. If smoke tests fail, build is rejected — no point testing further. Also called "Build Verification Testing."

Sanity Testing: Narrow, focused testing of specific functionality after a bug fix or small change. Verifies that the fix works and didn't break related areas. A subset of regression.

Memory trick: Smoke = Is the machine on fire? (broad check). Sanity = Is THIS specific thing sane? (focused check).
52
What is Regression Testing and why is automation critical for it?
▼
Regression testing = re-running existing test cases after code changes (bug fixes, new features) to ensure previously working functionality still works.

Why automation is critical: In Agile, code changes happen daily. Manually running 500 regression test cases after every commit takes 2-3 days. With Selenium automation, same 500 tests run in 2-3 hours automatically. This is the #1 use case for Selenium in real companies — automated regression suite that runs on every code commit via Jenkins.
53
What is the difference between Retesting and Regression Testing?
▼
Retesting: Testing the SPECIFIC bug that was fixed to verify the fix works. Also called "confirmation testing." Only tests the defect that was reported.

Regression Testing: Testing all RELATED areas to ensure the fix didn't break anything else. Broader scope.

What is the difference between Interface and Abstract Class?🔥 HOT
▼
Interface: Only abstract method declarations (before Java 8). No constructor. No instance variables. A class can implement MULTIPLE interfaces. Keyword: implements. WebDriver is an interface.

Abstract Class: Can have both abstract AND concrete methods. Has constructor. Can have instance variables. A class can only extend ONE abstract class. Keyword: extends. BaseTest can be abstract class.

Why no multiple class inheritance? Diamond problem — if B and C both have method() and D extends both, ambiguous which to call. Interfaces solve this since Java forces you to implement/override.

What is method overloading vs method overriding?
▼
Overloading (compile-time polymorphism): Same method name, different parameters (different count or types) in the SAME class. Resolved at compile time by compiler.
void click() { } and void click(int x, int y) { }

Overriding (runtime polymorphism): Child class provides its OWN implementation of a parent class method. Same name, same signature. Resolved at RUNTIME.
ChromeDriver overrides WebDriver.get(url)

Key rule: You can overload static methods but cannot override them. You cannot override final methods.

What is == vs .equals() in Java?🔥 HOT
▼
== compares references — are both variables pointing to the SAME object in memory?
.equals() compares content — do the objects have the same VALUE?

Example:
String a = "hello"; String b = "hello"; → a == b may be true (string pool) but unreliable
String a = new String("hello"); String b = new String("hello"); → a == b is FALSE (different objects), a.equals(b) is TRUE

In Selenium: Always use Assert.assertEquals(actual, expected) which internally uses .equals(). Never use == for String comparison.

What is String immutability? What's the difference between String, StringBuilder, StringBuffer?
▼
String immutability: Once a String object is created, it cannot be changed. str = str + "world" creates a NEW String object, doesn't modify the original. The old object becomes eligible for GC.

StringBuilder: Mutable String — changes happen in the same object. Not thread-safe. Fast. Use for single-threaded string building (dynamic XPath building).

StringBuffer: Mutable + Thread-safe (synchronized methods). Slower. Use in multi-threaded environments.

Performance: String < StringBuffer < StringBuilder (for heavy string manipulation)

• final method = cannot override in subclass
• final class = cannot extend (String is final)

finally: block that always executes after try-catch, regardless of exception or not. Perfect place for driver.quit() — ensures browser closes even if test fails.

What is Page Object Model? Why is it used?🔥 HOT

What is @CacheLookup? When should you NOT use it?
▼
@CacheLookup caches the WebElement after first find — subsequent uses don't search DOM again. Makes tests faster.

Use for: Static elements that never change after page load (nav menu, logo, static headers).

Do NOT use for:
• Dynamic elements (appear/disappear via AJAX)
• Elements inside tables that refresh
• Elements loaded lazily
• After page navigation

What is the order of TestNG annotation execution?🔥 HOT

@BeforeSuite — once before ALL tests in the suite
@BeforeTest — once before each <test> tag in testng.xml
@BeforeClass — once before first @Test in the class
@BeforeMethod — before EACH @Test method
@Test — the actual test
@AfterMethod — after EACH @Test method
@AfterClass — once after last @Test in the class
@AfterTest — once after each <test> tag
@AfterSuite — once after ALL tests

Why use @BeforeClass instead of @BeforeMethod for browser setup?
▼
@BeforeMethod runs before EACH @Test method. If you have 10 tests, browser opens 10 times and closes 10 times. Slow, resource-intensive.

@BeforeClass runs ONCE before ALL tests in the class. Browser opens once. All 10 tests share the same browser session. Browser closes once at end.

When to use @BeforeMethod: When tests are truly independent and cannot share state (e.g., login/logout tests where each needs fresh session).

When to use @BeforeClass: When tests share a common starting state and can share browser (e.g., tests on same page, sequential user flows).

What is REST? What are its constraints?🔥 HOT
▼
REST (Representational State Transfer) = architectural style for web APIs using HTTP protocol.

REST constraints:
1. Stateless — each request is independent, server doesn't store client state
2. Client-Server — separate client and server concerns
3. Uniform Interface — standard HTTP methods (GET/POST/PUT/DELETE)
4. Cacheable — responses can be cached
5. Layered System — client doesn't know if connecting directly to server

What is the difference between PUT and PATCH?🔥 HOT
▼
PUT = Replace the ENTIRE resource. If you send only {"name":"John"} in a PUT for user with id=1, all other fields (email, phone, age) become null/empty. Full replacement.

PATCH = Update PARTIAL resource. If you send only {"name":"John"} in PATCH, only name changes — email, phone, age remain unchanged. Partial update.

PUT is idempotent — calling it multiple times with same data has same effect. PATCH may or may not be idempotent depending on implementation.

What are path parameters vs query parameters?
▼

URL: /users/{id} → /users/5

URL: /universities?country=India&limit=10
REST Assured: given().queryParam("country","India").queryParam("limit","10").when().get("/universities")

Rule: Path params = "which resource?". Query params = "which subset/format of that resource?"

What is the difference between WHERE and HAVING?🔥 HOT
▼
WHERE: Filters individual ROWS before grouping. Cannot use aggregate functions (COUNT, SUM, AVG). Applied before GROUP BY.

HAVING: Filters GROUPS after GROUP BY. CAN use aggregate functions. Applied after GROUP BY.

Example: Departments with more than 2 employees hired after 2019:
SELECT dept_id, COUNT(*) FROM employees
WHERE hire_date > '2019-01-01' — filter rows first
GROUP BY dept_id
HAVING COUNT(*) > 2; — filter groups after

WHERE = row filter. HAVING = group filter.

LEFT JOIN: ALL rows from left table + matching from right. Non-matching right = NULL. Good for "employees with OR without departments."

RIGHT JOIN: ALL rows from right + matching from left. Non-matching left = NULL.

FULL OUTER JOIN: ALL rows from both. NULL where no match either side. MySQL: LEFT JOIN UNION RIGHT JOIN.

SELF JOIN: Table joined with itself. Employee-manager: e LEFT JOIN e m ON e.manager_id = m.emp_id

Write SQL to find the 2nd highest salary.🔥 HOT

What is the difference between RANK, DENSE_RANK, ROW_NUMBER?🔥 HOT
▼
All are window functions, all require OVER clause.

RANK(): Same values get same rank, next rank SKIPS numbers. Ties: 1, 2, 2, 4 (skips 3)

DENSE_RANK(): Same values get same rank, NO skipping. Ties: 1, 2, 2, 3

ROW_NUMBER(): Unique sequential number regardless of ties. Always: 1, 2, 3, 4 (even if salaries are equal)

Use DENSE_RANK for "Nth highest" problems. Use ROW_NUMBER for deduplication (keep one row per group). Use RANK when you want "tied position" visibility.

What is normalization? Explain 1NF, 2NF, 3NF.🔥 HOT
▼
Normalization = organizing database tables to reduce redundancy and improve data integrity.

1NF: Each column has atomic (indivisible) values. No repeating groups. No arrays in cells. Each row is unique. Violation: storing "phone1, phone2" in one column.

2NF: Must be 1NF + no partial dependency. Every non-key column depends on the ENTIRE primary key (not just part of composite key).

3NF: Must be 2NF + no transitive dependency. Non-key column should depend ONLY on primary key, not on another non-key column. Violation: storing city based on zip_code in orders table.

What are ACID properties?🔥 HOT
▼
ACID = properties that guarantee reliable database transactions:

A — Atomicity: All or nothing. Bank transfer: debit AND credit both happen, or neither. Can't have partial transaction.

C — Consistency: Database moves from one valid state to another. Rules/constraints always maintained. Can't violate foreign key constraints.

I — Isolation: Concurrent transactions don't interfere with each other. Each sees consistent snapshot. Prevents dirty reads.

D — Durability: Once committed, data is permanently saved even if system crashes. Data persists.

What is a subquery? What is a correlated subquery?
▼
Subquery: Query nested inside another query. Inner executes first, result used by outer.
SELECT name FROM employees WHERE salary > (SELECT AVG(salary) FROM employees);

Correlated Subquery: Inner query references a column from the OUTER query. Executes once per row of outer query — slower but powerful.

— finds employees earning above their OWN department average. The inner query uses e.dept_id from the outer query, so it runs fresh for each row.

What is the difference between UNION and UNION ALL?
▼
UNION: Combines results of two SELECT statements and REMOVES duplicate rows. Slower (requires sorting/deduplication).

UNION ALL: Combines results and KEEPS all rows including duplicates. Faster (no dedup step).

Both require: same number of columns, compatible data types, columns in same order.

Use UNION when you need unique results (e.g., all customers who ordered in Jan OR Feb — don't want same customer twice). Use UNION ALL when duplicates are acceptable/expected or for performance.

What is EXISTS vs IN? Which is faster?
▼
IN: Checks if value is in a predefined list or subquery result set.
WHERE dept_id IN (SELECT id FROM departments WHERE location='Chennai')

EXISTS: Checks if a subquery returns any rows. Returns TRUE on first match and stops.
WHERE EXISTS (SELECT 1 FROM departments WHERE id=e.dept_id AND location='Chennai')

Performance: EXISTS is generally faster for large datasets because it stops at first match. IN loads all subquery results into memory then checks. EXISTS is better when checking existence only, IN is better for small static lists.

What is a Primary Key vs Unique Key vs Foreign Key?
▼
Primary Key: Uniquely identifies each row. Cannot be NULL. Cannot have duplicates. Only ONE per table. Creates clustered index automatically. Example: emp_id, student_id.

Unique Key: All values must be unique. CAN have ONE NULL. Multiple per table. Creates non-clustered index. Example: email column — each person has unique email.

Foreign Key: Column referencing the Primary Key of another table. Enforces referential integrity — can't insert FK value that doesn't exist in parent PK. CAN be NULL. CAN have duplicates. Example: dept_id in employees referencing id in departments.

What is indexing? When should you create an index?
▼
Index = separate data structure that speeds up SELECT queries at cost of storage and slower INSERT/UPDATE/DELETE.

Like book index — instead of reading all pages, jump directly to right page.

Create index when:
• Column used frequently in WHERE clause
• Column used in JOIN conditions
• Column used in ORDER BY
• Large tables with frequent reads

Don't create index when:
• Small tables (full scan faster)
• Columns with many NULLs
• Tables with frequent inserts (index slows inserts)
• Columns rarely used in queries

Primary key auto-creates clustered index.

What is a transaction? Explain COMMIT, ROLLBACK, SAVEPOINT.
▼
Transaction = group of SQL operations treated as ONE unit — all succeed or all fail (atomicity).

START TRANSACTION;
UPDATE accounts SET balance = balance - 1000 WHERE id = 1; -- debit
UPDATE accounts SET balance = balance + 1000 WHERE id = 2; -- credit
COMMIT; -- make permanent
-- OR ROLLBACK; -- undo both changes

SAVEPOINT: Intermediate checkpoint within a transaction. Can rollback to savepoint without undoing entire transaction. Like a save point in a game.
SAVEPOINT sp1; → ROLLBACK TO sp1; — undo only after savepoint

Write SQL: top 3 earners per department using window functions.
▼
SELECT name, salary, dept_id FROM (
SELECT name, salary, dept_id,
DENSE_RANK() OVER (
PARTITION BY dept_id
ORDER BY salary DESC
) AS rnk
FROM employees
) ranked
WHERE rnk <= 3;

PARTITION BY dept_id = rank within each department separately. ORDER BY salary DESC = highest salary = rank 1. WHERE rnk <= 3 = keep only top 3 per department. Using DENSE_RANK so tied salaries get same rank.
